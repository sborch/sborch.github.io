Overview

dbView is a query and analysis tool that is intended provide a consistent environment in which to work with different types of databases.

I created and maintain dbView to support my day-to-day work and for this reason dbView is continually evolving and changing as my needs change.

Please see me with any questions and/or for an overview of dbView capabilities.

SQLite database library

I rely heavily on SQLite (SQLite Home Page) as a versatile database engine for much of the underpinnings of dbView and my related data-related tools. SQLite is a software library that implements a self-contained, serverless, zero-configuration, transactional SQL database engine.

Note that SQLite uses dynamic typing rather than the static typing we are used to with SQL Server and other traditional RDBMS engines.
Itâ€™s a bit different but very flexible and can be especially useful for working with text files and ad-hoc data analysis across multiple data sources. For this reason you should review the SQLite documentation on datatypes at Datatypes In SQLite.

Also if you are interested, I use an ADO.NET provider for SQLite called System.Data.SQLite. This library allows for easy integration of SQLite functionality into a .NET application.

Important Note - Connecting to SQLite Databases

If you want the connection to fail if the specified database file does NOT exist, then set FailIfMissing=True

SQLITE:
;Data Source="C:\Temp\MyDatabase.db"
;Foreign Keys=True
;Read Only=False
;FailIfMissing=True

Otherwise if you want the connection to always succeed, then set FailIfMissing=False. This will cause a new empty SQLite database file to be automatically created if the specified file does not already exist.

SQLITE:
;Data Source="C:\Temp\MyDatabase.db"
;Foreign Keys=True
;Read Only=False
;FailIfMissing=False
