Exporting Data

Open any supported database and run one of the following export command (replace example parameter values with your own values)


Option 1 - Exporting to Delimited Text File

export

  tgt_type                 = 'text_delimited'
  tgt_file                 = 'REPLACE_ME_Target_File_Name'

  tgt_file_field_separator = 'tab'
  tgt_file_text_qualifier  = 'none'
  tgt_datetime_format      = 'yyyy-MM-dd HH:mm:ss.fff'

select * from dbo.Table1;
parameters type required description/notes
tgt_type string required valid values are 'text_delimited' or 'sql_script'
tgt_file string required fully pathed name of the target file
tgt_file_field_separator string required must be either ‘tab’ or a single character

e.g. ',' or ';'
tgt_file_text_qualifier string optional the character used to specify where a text value begins and ends.

valid values are: 'none', 'single-quote', 'double-quote'.

default is 'none' if not defined
tgt_datetime_format string optional .NET datetime format string to use when writing datetime values to file

default value is 'yyyy-MM-dd HH:mm:ss.fff' if not defined


Option 2 - Exporting to SQL Insert Script

export

  tgt_type                 = 'sql_script'
  tgt_file                 = 'REPLACE_ME_Target_File_Name'

  tgt_table                = 'dbo.MyNewTable'
  tgt_include_field_names  = 'true'

select * from dbo.Table1;
parameters type required description/notes
tgt_type string required valid values are 'text_delimited' or 'sql_script'
tgt_file string required fully pathed name of the target file
tgt_table string required name of table to use in sql insert statements
tgt_include_field_names string optional ‘true’ or ‘false’

defines whether a field name list is built for the insert sql statements.

    i.e. insert into <table> (f1, f2, f3) values (v1,v2,v3)

no field name list means that the file will be smaller but the inserts will then depend on the same field order in the target table.

    i.e. insert into <table> values (v1,v2,v3)

default value is ‘true’ if not defined
